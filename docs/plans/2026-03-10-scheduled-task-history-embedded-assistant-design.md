# 计划任务历史弹窗嵌入助手页设计

**目标:** 在计划任务的历史记录弹窗右侧嵌入一个完整的 AI 助手对话页面实例，用当前运行记录关联的会话初始化，并复用现有助手页能力。

**范围:**
- 修改历史记录弹窗右侧内容区域。
- 为助手页增加嵌入模式。
- 让嵌入模式支持按指定会话初始化。

**非目标:**
- 不引入真正的 iframe 或独立浏览器页面。
- 不重写消息区、输入区、工作区抽屉等助手核心组件。
- 不支持在嵌入模式切换助手或切换其他会话。

**设计决策:**
- 新增一个嵌入式助手容器组件，在同一个 Vue 应用内挂载 `AssistantPage`。
- 为 `AssistantPage` 增加 `embedded` 模式及初始化会话参数。
- 嵌入模式隐藏左侧助手/会话侧栏，不更新主标签页标题和图标。
- 历史弹窗右侧仅传递当前运行记录对应的 `conversationId`，会话数据继续走助手页原有加载链路。
- 若当前运行记录没有关联会话，则显示空态，不挂载助手实例。

**数据流:**
- 用户在历史弹窗左侧切换运行记录。
- 弹窗拿到当前运行记录详情中的 `conversation.id`。
- 右侧嵌入助手实例用该 `conversationId` 初始化并加载完整会话。

**风险与验证:**
- 需要避免嵌入实例和普通助手标签页在导航、副作用和事件订阅上互相污染。
- 通过本地构建和针对历史弹窗的手动交互验证确认嵌入模式可正常加载、发送消息和展开工作区。
